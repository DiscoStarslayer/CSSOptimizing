Optimizing CSS Animations

Browser Rendering Pipeline

  • Any time you change what is displayed in the browser, it runs this rendering pipeline
  • Computationally Heavy
  • Layouting is not deterministic, needs to be run proceduraly

Javascript Pipeline

  • Add / Remove / Modify DOM elements
  • Not just Javascript
  • Modifying CSS Properties
  • CSS Transitions

Styling Pipeline

  • Match CSS Rules to selectors
  • Computes all styles for each element
  • Complicated CSS and wide reaching styles like `*` can slow this down

Layout Pipeline

  • Use computed rules to determine size of elements and where to place them
  • Has to be rendered proceduraly, nature of CSS means parent and children elements can effect styles of the element being rendered
